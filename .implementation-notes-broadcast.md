# Broadcast Groups Implementation Notes

## Current Schema
- `groups` table has: id, code, name, description, type (enum: "private", "specialist"), creatorId, image, memberCount, createdAsRole, createdAt, updatedAt
- `groupMembers` table has: id, groupId, userId, role (enum: "admin", "member", "creator", "follower"), joinedAt
- `groupInvites` table has: id, groupId, inviterId, inviteeId, status (enum: "pending", "accepted", "rejected")

## Changes Needed

### 1. Schema Changes
- Add "broadcast" to groups.type enum: `["private", "specialist", "broadcast"]`
- Add fields to groups: `linkedEntityId` (int, nullable - estab ID or user ID), `linkedEntityType` (enum: "establishment", "specialist", "critic"), `isFixed` (boolean, default false - cannot be deleted by owner)
- Add fields to groupMembers: `hidden` (boolean, default false - for "Ocultar"), `leftAt` (timestamp, nullable - for "Sair")

### 2. Auto-creation of Broadcast Groups
- When an establishment is created → auto-create broadcast group with name = estab name, linkedEntityId = estab.id, linkedEntityType = "establishment", isFixed = true
- When a user becomes specialist → auto-create broadcast group with name = user display name, linkedEntityId = user.id, linkedEntityType = "specialist", isFixed = true
- When a user becomes critic → auto-create broadcast group with name = user display name, linkedEntityId = user.id, linkedEntityType = "critic", isFixed = true

### 3. Auto-join Logic
- `toggleSave` endpoint (line 1809): when saved=true, also join the broadcast group of that establishment
- `followSpecialist` endpoint (line 2663): also join the broadcast group of that specialist
- `followUser` endpoint (line 3347): if target user is specialist/critic, also join their broadcast group

### 4. Key Endpoints to Modify
- `toggleSave` (line 1809): hook auto-join to broadcast group
- `followSpecialist` (line 2663): hook auto-join to specialist broadcast group
- `followUser` (line 3347): hook auto-join to specialist/critic broadcast group
- `groups.follow` (line 1582): existing follow group logic
- `groups.unfollow` (line 1591): existing unfollow logic

### 5. Frontend Changes
- CreateGroupModal: update options (Privado + Grupo de Transmissão for business/specialist/critic)
- "Seguindo" tab: add 3 sub-tabs (Grupos de quem sigo, Grupos de Transmissões, Grupos Ocultos)
- Add "Sair deste Grupo" and "Ocultar este grupo" buttons for user/critic/specialist
- Business can only send messages if they have Business Pro plan

### 6. Permission Rules
- Broadcast group messages: only the linked entity owner can send
- Business: needs Business Pro plan to send
- Specialist/Critic: can send freely (already paid plan)
- Fixed broadcast groups: only support/admin/owner can delete

### 7. Existing Hooks
- `followBusiness(estabId, userId)` already called in toggleSave when saved=true
- `unfollowBusiness(estabId, userId)` already called in toggleSave when saved=false
- `followSpecialist(userId, specialistId)` at line 2663
- `followUser(userId, targetId)` at line 3347
