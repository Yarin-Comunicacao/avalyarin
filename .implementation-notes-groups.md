# Implementação: Separar visão de grupos por role (Owner)

## Requisito
- Owner visualizando como critic/specialist/business NÃO vê grupos criados como user
- Owner visualizando como user vê apenas grupos de user
- Usuários REAIS que fizeram upgrade (user → specialist/critic) CONTINUAM vendo todos os seus grupos

## Abordagem
- Adicionar campo `createdAsRole` na tabela `groups` (varchar, default "user")
- Quando owner cria grupo visualizando como um role específico, salvar esse role
- No endpoint `myGroups`, se `effectiveRole` está presente (owner visualizando), filtrar por `createdAsRole`
- Se `effectiveRole` é null (usuário real ou owner sem viewingAs), retornar TODOS os grupos

## Arquivos a modificar
1. `drizzle/schema.ts` - adicionar campo `createdAsRole` 
2. `server/db-groups.ts` - getMyGroups aceitar effectiveRole param e filtrar; createGroup salvar createdAsRole
3. `server/routers.ts` - passar effectiveRole ao chamar getMyGroups

## Contexto técnico
- effectiveRole vem do header `x-viewing-as` (setado no OwnerViewContext → window.__ownerViewingAs → main.tsx headers())
- context.ts lê o header e expõe como ctx.effectiveRole
- Quando owner muda role, queryClient.invalidateQueries() refaz todas queries
